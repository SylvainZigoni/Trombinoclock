# User Stories

En tant que ..., je veux ... afin de ....

Exemple d'un site ecommerce :
En tant que client, je veux pouvoir accéder à une page produit, afin de consulter les détails du produit et éventuellement l'ajouter à mon panier.
En tant qu'administrateur, je veux pouvoir gérer les commandes, afin de suivre les ventes et traiter les expéditions.

Les users stories permettent de définir les besoins fonctionnels du projet de manière claire et concise, en se concentrant sur les objectifs des utilisateurs finaux.

C'est un document qui va être partagé avec le client pour validation avant de commencer le développement.
